# GoIT Algorithm Homework 06: Графи та алгоритми пошуку

## Опис
У цьому домашньому завданні реалізовано моделювання транспортної мережі міста за допомогою графів, а також програмно реалізовано базові алгоритми пошуку шляхів (DFS, BFS) та пошуку найкоротшого шляху у зваженому графі (алгоритм Дейкстри).

## Завдання 1: Моделювання та візуалізація
Було створено граф, що моделює схему метрополітену з пересадками.
- **Вершини (Nodes):** Станції метро.
- **Ребра (Edges):** Перегони між станціями та пересадочні вузли.
- **Ваги (Weights):** Середній час проїзду між станціями у хвилинах.

Граф візуалізовано за допомогою бібліотеки `matplotlib`. Вершини відображають станції, а лінії — зв'язки між ними.

## Завдання 2: Порівняння алгоритмів DFS та BFS
Для аналізу роботи алгоритмів було обрано маршрут:
- **Початок:** Станція "Університет"
- **Кінець:** Станція "Печерська"

### Результати виконання

**1. BFS (Breadth-First Search — Пошук у ширину)**
- **Принцип роботи:** Алгоритм обходить граф "хвилями" або шарами. Спочатку він перевіряє всіх безпосередніх сусідів стартової вершини, потім сусідів сусідів і так далі.
- **Результат:** BFS знайшов шлях: `['Університет', 'Театральна', 'Золоті Ворота', 'Палац Спорту', 'Кловська', 'Печерська']`.
- **Висновки:** Оскільки граф незважений (для цього алгоритму ми ігноруємо вагу ребер), BFS гарантовано знаходить **найкоротший шлях за кількістю ребер** (мінімальна кількість пересадок/зупинок).

**2. DFS (Depth-First Search — Пошук у глибину)**
- **Принцип роботи:** Алгоритм обирає один із сусідніх вузлів і рухається вглиб по цій гілці наскільки це можливо, перш ніж повернутися назад (backtracking), якщо зайде в глухий кут або досягне цілі.
- **Результат:** DFS знайшов шлях, який відрізняється від BFS (наприклад, через "Хрещатик" та "Арсенальну", або інший довший маршрут).
- **Висновки:** Шлях, знайдений DFS, не є найкоротшим. Алгоритм не шукає оптимальність, його мета — перевірити наявність шляху. Тому він може "блукати" лабіринтом графа, перш ніж потрапити в цільову точку.

### Чому шляхи відрізняються?
Різниця полягає в механізмі черги обробки вершин:
- **BFS** використовує чергу (`FIFO`), що забезпечує рівномірний пошук у всіх напрямках.
- **DFS** використовує стек (`LIFO`) (або рекурсію), що змушує його йти до кінця обраного маршруту, ігноруючи коротші альтернативи на розгалуженнях.

## Завдання 3: Алгоритм Дейкстри
Алгоритм Дейкстри було використано для знаходження найкоротшого шляху у **зваженому графі**, де вага ребра — це час проїзду.

- На відміну від BFS, який шукає шлях з найменшою кількістю зупинок, Дейкстра шукає шлях з **найменшим сумарним часом**.
- У нашій моделі цей алгоритм розрахував оптимальний маршрут, враховуючи, що пересадки та довгі перегони займають різний час. Це найбільш точний метод для реальних навігаційних задач.
